## Core Settings

### Global vs Per-Repo Config
- Global: Default rules for all repos
- Per-Repo: Override rules for specific repos
- Per-Directory: Override rules for specific directories (ideal for monorepos, [learn more](/how_to_use/en/code_review/configs/directory_level))

### Config Priority

### Base Branches

## Review Behavior

### Automated Review

Controls whether Kody automatically reviews PRs when first opened, or only runs when you comment `@kody start-review`.

<Info>
  This setting only affects the **initial review** when a PR opens. For subsequent commits, see Review Cadence below.
</Info>

#### Review Cadence

Controls how Kody handles reviews on **subsequent commits** after the PR is already open.

<Tabs>
  <Tab title="Automatic">
    Review every new push to the PR. You get continuous feedback on all changes.
  </Tab>
  
  <Tab title="Auto-pause">
    Pause reviews when detecting rapid consecutive pushes (e.g., 3 pushes in 15 minutes). Prevents spam during active coding. Reviews resume automatically after the configured cooldown period.
    
    Configuration:
    - **Pushes to trigger pause**: Default 3
    - **Time window**: Default 15 minutes
  </Tab>
  
  <Tab title="Manual">
    Only run follow-up reviews when you comment `@kody start-review`. The initial review (controlled by "Automated Review" above) is independent.
  </Tab>
</Tabs>

### PR Workflow

#### Auto-approve PRs
Kody approves PRs automatically when no issues are found.

Platform support: GitHub, GitLab, Bitbucket

Use for low-risk changes (docs, tests) or teams with strong CI/CD.

#### Request Changes
Kody requests changes when finding **critical issues**.

Platform support: GitHub, Bitbucket (not GitLab yet)

Use to enforce security standards or prevent breaking changes.

### Skip Conditions
- Ignored Files: Use glob patterns (e.g., `**/*.js`), one per line
- Ignored Titles: Skip PRs with specific keywords (case-insensitive, max 100 chars)

## Analysis Types

### Code Quality
- Security: SQL injection, XSS, security threats
- Performance: Caching, query optimization, speed improvements
- Error Handling: Exception management, error messages
- Potential Issues: Null pointers, resource leaks, infinite loops

### Code Structure
- Refactoring: Code organization, function size, duplication
- Maintainability: Future-proof code patterns
- Code Style: Consistent formatting and standards
- Documentation: Clear comments and API docs
- Kody Rules: Custom rule enforcement

## Ignored Files

Skip specific files from review:

- Use glob patterns (e.g., `**/*.js`)
- One pattern per line
- Default ignores: `yarn.lock`, `package-lock.json`, `package.json`, `.env`

## Ignore Title Keywords

Skip review for PRs with specific titles (case-insensitive, max 100 chars per keyword)
