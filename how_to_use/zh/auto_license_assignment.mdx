---
title: "自动许可证分配"
---

自动许可证分配功能简化了组织内开发人员许可证的管理。Kodus 可以自动为活跃贡献者分配许可证,而不是手动为每个开发人员分配许可证。

## 工作原理

启用后,Kodus 会监控代码库中的拉取请求活动。

-   **自动分配:**如果用户创建了多个拉取请求,Kodus 会检查组织中是否有可用的许可证槽位。
    -   如果有可用槽位,用户将自动分配许可证,并审查他们的 PR。
    -   如果没有可用槽位,Kodus 将对拉取请求做出反应,对于 Github 使用点踩表情符号(👎),对于 Gitlab 使用挂锁表情符号(🔒),以表明由于缺少许可证而无法执行审查。

## 忽略的用户

您可以配置**忽略的用户**列表。这些用户将被排除在自动分配流程之外。

-   **不审查:**Kodus 不会审查忽略用户创建的拉取请求。
-   **不反应:**即使超过 PR 限制,Kodus 也不会对他们的 PR 做出反应。就像 Kodus 完全忽略了 PR 一样。

这对于机器人、服务账户或不需要代码审查的特定团队成员很有用。

## 配置

要启用和配置自动许可证分配:

1. 转到您的**代码审查设置**。
2. 导航到**订阅**选项卡。
3. 找到**自动分配许可证**部分。
    - 使用切换开关启用或禁用该功能。
    - 如果启用,您可以选择用户添加到**忽略的用户**列表。
