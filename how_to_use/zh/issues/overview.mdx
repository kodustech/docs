## 快速概述

- **自动跟踪**:自动捕获已关闭 PR 中未实施的建议
- **智能解决**:当建议在未来的 PR 中实施时自动解决
- **高级过滤**:按状态、严重性和类别保存自定义视图
- **零设置**:自动与您现有的代码审查工作流程配合使用

## 是什么

**Kody 工单**功能自动跟踪 Kody 生成的所有在您的拉取请求(PR)中**未实施**的建议。它就像声纳一样,提供待处理改进的实时列表,确保您永远不会失去对关键反馈的关注。

## 工作原理

### 1. 工单创建

<Note>
  工单仅在 PR **关闭**时创建 - 不是在它们打开或审查期间。
</Note>

<Warning>
  类型为 **PR** 的 Kody 规则不会生成工单。只有文件级和特定代码的建议才会创建可跟踪的工单。
</Warning>

- 当 PR **关闭**时,Kody 汇编该 PR 中所有**未应用**的建议
- 对于每个未实施的建议,Kody 自动创建一个链接到代码库的**工单**
- 每个工单都包含原始建议上下文、文件位置和严重性级别

### 2. 自动解决

- 如果开发人员在**后续 PR** 中实施了建议的更改,Kody 会将相关工单标记为**已解决**
- Kody 使用代码分析智能地将实施的更改与现有工单匹配
- 解决自动发生,无需手动干预

### 3. 手动管理

#### 状态控制

- **打开**:需要关注的新工单
- **已解决**:当建议被实施时自动标记
- **已忽略**:当建议与您的团队不相关时手动标记

#### 严重性调整

- 每个工单都有严重性级别(`严重`、`高`、`中等`、`低`)
- 您可以随时更新严重性级别以反映团队的优先级
- 使用严重性首先关注最具影响力的改进

### 4. 高级过滤

<Warning>
  过滤器是强大的工具 - 保存自定义视图以快速访问对您的工作流程最重要的工单。
</Warning>

- **状态过滤器**:打开、已解决、已忽略
- **严重性过滤器**:严重、高、中等、低
- **类别过滤器**:安全性、性能、代码风格等
- **代码库过滤器**:专注于特定项目
- **保存自定义视图**:自动应用您首选的过滤器

## 为什么重要

### 对于开发团队

- **永远不会失去跟踪**重要的代码改进
- **根据实际影响和严重性优先处理工作**
- **从常见的已忽略 vs. 已实施建议中学习模式**

### 对于工程经理

- **跟踪代码质量**趋势跨代码库
- **识别瓶颈**在建议实施中
- **衡量改进**在代码审查跟进中
