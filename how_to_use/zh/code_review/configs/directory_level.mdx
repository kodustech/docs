## 概述

为代码库中的特定文件夹应用自定义代码审查设置。非常适合不同区域有不同标准或所有权的单体代码库。

Kody 为每个 PR 选择单个配置:**目录** → **代码库** → **全局**

<Note>
  了解设置如何在级别之间流动,请参阅<a href="/how_to_use/en/code_review/configs/inheritance_overrides">配置继承与覆盖</a>。
</Note>

## 工作原理

- 在任何深度配置任何文件夹(例如 `api/src/services`)
- 根据需要配置任意多个文件夹
- **不重叠**:如果配置 `api/src/services`,则不能同时配置 `api/src/services/users`

### 选择规则

- **触及一个已配置目录**:使用该目录的配置
- **未触及已配置目录**:使用代码库配置(如果没有代码库配置则使用全局配置)
- **触及两个或更多已配置目录**:升级到代码库配置(如果没有代码库配置则使用全局配置)

<Note>
  Kody 对每个 PR 使用一个配置以保持审查可预测。混合配置会导致冲突。
</Note>

## 示例

<Tabs>
  <Tab title="单个已配置目录">
    您配置了 `src/microservice-alpha`。PR 更改了 `src/microservice-alpha` 和未配置文件夹中的文件。

    使用 **microservice-alpha** 配置。
  </Tab>
  <Tab title="无已配置目录">
    PR 仅触及没有特定配置的目录。

    使用**代码库**配置(如果没有代码库配置则使用**全局**配置)。
  </Tab>
  <Tab title="多个已配置目录">
    您配置了 `src/microservice-alpha` 和 `src/microservice-beta`。PR 触及两者。

    升级到**代码库**配置(如果没有代码库配置则使用**全局**配置)。
  </Tab>
</Tabs>

## 设置

1. 打开代码库的代码审查设置
2. 添加目录条目并分配配置
3. 保存更改

如果您还使用 `kodus-config.yml`,它会覆盖网页设置(请参阅通用设置中的配置优先级)。
