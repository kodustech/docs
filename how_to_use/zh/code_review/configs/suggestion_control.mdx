## 建议限制

### 限制类型

- 按 PR(默认):整个拉取请求中的最大建议数
- 按文件:每个文件的最大建议数

### 最大建议数

设置 Kody 可以提出多少建议:

- 最小值:6 条建议
- 默认值:10 条建议
- 设置为 0 表示无限制

专业提示:从 10 开始,根据团队反馈进行调整。太多建议可能会让人不知所措,太少可能会遗漏重要问题。

## 严重性级别

### 最低严重性

选择 Kody 将报告哪些问题:

- 低/全部:风格问题、小改进
- 中等:常见最佳实践、潜在错误
- 高:重大问题、明确的错误
- 严重:安全问题、主要问题

## 建议分组

### 统一评论(默认)

<img src="https://app.kodus.io/_next/image?url=%2Fassets%2Fimages%2Ffull_grouping.png&w=1080&q=75" alt="统一评论" />
将类似问题合并到一条评论中,显示所有受影响的文件和行。适用于:

- 有重复问题的大型 PR
- 保持讨论集中
- 减少通知噪音

### 单独评论

<img src="https://app.kodus.io/_next/image?url=%2Fassets%2Fimages%2Fminimal_grouping.png&w=1080&q=75" alt="单独评论" />
每个问题单独评论。更适合以下情况:

- 您需要直接链接到每个位置
- 团队更喜欢集中、特定行的讨论
