## 概述

Kodus 配置跨三个级别工作:**全局**、**代码库**和**目录**。

<Note>
最具体的级别优先:目录 > 代码库 > 全局
</Note>

```
全局
 ├─ 代码库 A
 │   ├─ 目录 A1
 │   └─ 目录 A2
 └─ 代码库 B
```

## 工作原理

字段默认从父级继承。当您在全局级别更改字段时,所有没有覆盖的代码库和目录都会自动更新。

<Info>
覆盖是按字段进行的。您可以覆盖一个字段并继承其他所有字段。
</Info>

一旦被覆盖,字段将停止跟随父级并使用其本地值。

## 示例

<Tabs>
  <Tab title="全局更改传播">
    您在全局级别更改**严重性阈值**。

    所有没有覆盖的代码库和目录都会自动更新。已覆盖的保持不变。
  </Tab>

  <Tab title="代码库覆盖">
    在代码库 X 中,您覆盖**审查开始消息**。

    代码库 X 内的目录现在从代码库 X 继承(除非它们有自己的覆盖)。
  </Tab>

  <Tab title="返回继承">
    在代码库 Y 中,您删除**活动类别**的覆盖。

    该字段再次跟随全局并接收未来的全局更改。
  </Tab>
</Tabs>

## 最佳实践

- 从全局开始设置强大的默认值
- 仅在该代码库/目录有特定需求时才覆盖

## 常见问题

<AccordionGroup>
  <Accordion title="如果我在全局级别更改某些内容,会破坏我的本地调整吗?">
    不会。本地覆盖保持不变。
  </Accordion>

  <Accordion title="我可以只覆盖一个字段并继承其余字段吗?">
    可以。覆盖是按字段进行的。
  </Accordion>

  <Accordion title="目录是从代码库还是全局继承?">
    首先从代码库继承。如果代码库对该字段没有覆盖,则从全局继承。
  </Accordion>
</AccordionGroup>
