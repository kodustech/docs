## 快速概述

- **自动生成**:Kody 分析您的审查历史以创建新规则
- **数据驱动**:使用 3 个月的审查数据获取智能见解
- **智能建议**:基于实际团队模式生成规则
- **需要手动导入**:生成的规则必须手动审查和导入才能生效
- **持续学习**:随着团队发展每周更新规则

## 工作原理

### 1. 访问规则生成

导航到您的代码库设置:

1. 转到**代码审查设置** → **代码库** → **Kody 规则**
2. 查找**"生成 Kody 规则"**按钮
3. 点击开始自动生成过程

### 2. 分析过程

当您点击"生成 Kody 规则"时,Kody 将:

- **扫描审查历史**:分析代码库中最近 3 个月的代码审查
- **识别模式**:发现常见的反馈主题和编码偏好
- **提取标准**:将审查模式转换为可操作的规则
- **生成建议**:创建针对团队风格定制的自定义规则建议

### 3. 审查生成的规则

分析完成后:

1. 将出现**"查看新规则"**按钮
2. 点击查看生成的规则建议
3. 浏览建议的规则及其解释
4. **重要**:规则尚未激活 - 此时它们只是建议

### 4. 导入规则(激活所需)

<Warning>
  **关键步骤**:在您手动导入之前,生成的规则不会生效。
</Warning>

要使规则生效:

- **选择性选择**:仅导入符合团队需求的规则
- **自定义**:在导入前根据需要修改规则
- **应用**:将选定的规则添加到活动规则集
- **激活**:只有导入的规则才会在代码审查期间执行

## 持续学习

初始设置后,Kody 维护持续的学习循环:

- **每周更新**:Kody 每周生成新的规则建议
- **演变标准**:规则建议随着团队的编码风格演变而调整
- **精确度提高**:建议随着时间的推移变得更加精确
- **定期审查**:每周出现新的规则建议供您审查

## 要求

- **最少数据**:代码库中至少 3 个月的审查历史
- **活跃审查**:定期代码审查活动以更好地检测模式
- **代码库访问**:修改代码库规则的适当权限
