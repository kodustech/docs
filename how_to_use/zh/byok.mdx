BYOK(自带密钥)为您提供对 AI 模型使用的完全控制:选择任何模型,只为您使用的付费,并直接在提供商的仪表板中监控成本。

## 入门

<Steps>
  <Step title="访问 BYOK 设置">
    导航到您的组织设置 [app.kodus.io/organization/byok](https://app.kodus.io/organization/byok)
  </Step>

  <Step title="配置主模型">
    选择您的主要提供商(OpenAI、Google Gemini、Anthropic 等)并使用您首选的模型输入您的 API 密钥
  </Step>

  <Step title="配置后备模型(推荐)">
    设置辅助模型作为备份。如果您的主模型出现问题或不可用,审查会自动切换到后备模型
  </Step>

  <Step title="测试配置">
    Kodus 将在保存前验证 API 密钥和模型选择
  </Step>
</Steps>

<Info>
  **重要:**始终配置后备模型以确保即使主要提供商遇到停机或速率限制,也能持续进行代码审查。
</Info>

<Warning>
  保护您的 API 密钥安全,切勿共享。为 Kodus 使用单独的密钥以更好地跟踪使用和成本。
</Warning>

## 故障排除

<AccordionGroup>
  <Accordion title="无效的 API 密钥错误">
    **解决方案:**
    - 验证 API 密钥是否正确复制,没有多余的空格
    - 检查您的提供商账户是否启用了计费
    - 确保 API 密钥具有必要的权限
    - 如果问题持续存在,请尝试生成新的 API 密钥
  </Accordion>

  <Accordion title="未找到模型错误">
    **解决方案:**
    - 验证模型名称是否与提供商的确切格式匹配
    - 检查模型在您的地区是否可用
    - 确保您的账户可以访问指定的模型
    - 尝试不同的模型以测试问题是否特定于模型
  </Accordion>

  <Accordion title="速率限制问题">
    **解决方案:**
    - 检查您提供商的速率限制并根据需要升级
    - 考虑切换到速率限制更高的提供商
    - 联系您的提供商支持以增加限制
    - 如果允许,在多个 API 密钥之间分配使用
  </Accordion>

  <Accordion title="自定义 URL 连接问题">
    **解决方案:**
    - 验证基础 URL 格式包括协议(https://)
    - 检查端点是否支持 OpenAI 兼容 API
    - 使用 curl 或 Postman 等工具直接测试 URL
    - 确保没有防火墙限制阻止连接
  </Accordion>

  <Accordion title="意外的高成本">
    **解决方案:**
    - 在提供商仪表板中审查您的使用情况
    - 检查是否适当设置了速率限制
    - 考虑切换到更具成本效益的模型
    - 设置计费警报以尽早发现激增
  </Accordion>
</AccordionGroup>

## 常见问题

<AccordionGroup>
  <Accordion title="我可以随时切换提供商吗?">
    可以。更改立即对新审查生效。
  </Accordion>

  <Accordion title="如果我的 API 密钥用完积分会怎样?">
    审查会自动切换到您的后备模型(如果已配置)。没有后备,审查会失败,直到您添加积分。始终配置后备模型。
  </Accordion>

  <Accordion title="主/后备系统如何工作?">
    主模型默认处理所有审查。如果失败(速率限制、停机、错误),Kodus 会自动切换到后备。您只为处理每次审查的模型付费。
  </Accordion>

  <Accordion title="我应该对主和后备使用相同的提供商吗?">
    不应该。使用不同的提供商(例如,Google 主,OpenAI 后备)以防止特定于提供商的中断。
  </Accordion>

  <Accordion title="BYOK 与托管模式相比如何?">
    **BYOK ($10/开发/月):**您管理 API 密钥,完全控制,需要设置

    **托管 ($30/开发/月):**我们处理一切,固定成本,零配置
  </Accordion>

  <Accordion title="您是否安全存储我们的 API 密钥?">
    是的。密钥在静态和传输中加密。我们从不以明文记录或存储密钥。
  </Accordion>
</AccordionGroup>
