---
title: "我可以使用自己的 LLM 提供商进行代码审查吗"
description: "带上自己的 API 密钥并选择首选的 LLM 提供商用于 Kodus AI 代码审查。"
---

可以。Kodus 支持自带密钥（BYOK），让您使用首选的 LLM 提供商。

## 工作原理

1. 前往**设置** → **BYOK**（自带密钥）
2. 输入您首选提供商的 API 密钥
3. Kodus 使用您的密钥进行所有 AI 操作

## 为什么使用自己的提供商

- **数据控制** — 请求直接发送到您的提供商
- **成本管理** — 使用您现有的计费关系
- **模型选择** — 选择最适合您代码库的模型
- **合规性** — 满足 AI 工具使用的内部要求

## 自托管 LLM 提供商

如果您自托管 Kodus，还可以使用自托管 LLM 提供商：
- Novita
- Groq
- Together AI
- Fireworks AI

有关设置指南，请参阅 [Cookbook](/cookbook/en/novita)。

有关 BYOK 配置，请参阅[自带密钥](/how_to_use/en/byok)。
