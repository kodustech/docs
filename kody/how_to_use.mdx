## Using Kody via Communication Tools

If configured, you can interact with Kody directly through your communication tool. Although there may be some limitations in the UI, using Kody in this way makes it easier to ask questions and perform simple interactions.

### Slack

You can access Kody in two ways within Slack:

- **Direct Messages (DM)**: Use Kody in your Slack inbox for private queries or quick interactions.
- **Channel Integration**: Use Kody within a specific channel where it has been installed. This is useful for team-wide updates or discussions that require Kody's input.

### Discord

Similarly, Kody is available within Discord through two options:

- **Direct Messages (DM)**: Access Kody through your Discord inbox for one-on-one interactions. This is ideal for personal inquiries or when testing commands privately.
- **Channel Integration**: Use Kody within any designated channel where it has been installed. This option is helpful for team discussions and collaborative tasks where Kody’s input can provide insights or support.

## Example Interactions

Here are some specific commands to get you started with Kody in Slack or Discord:

- **Team Metrics and Productivity**:

  - `"@Kody, what are our team’s current metrics?"`
  - `"@Kody, are there any productivity issues affecting the team?"`
  - `"@Kody, do we have any overdue tasks this sprint?"`

- **Code Review**:

  - `"@Kody, please review this code snippet: [paste code]"`
  - `"@Kody, perform a code review on PR #123"`

- **Generate Documentation**:

  - `"@Kody, generate documentation for the class XYZ"`
  - `"@Kody, create documentation for this code: [paste code]"`
  - `"@Kody, document the new feature in [repository name]"`

- **Check Team Health and Flow**:

  - `"@Kody, what’s the current team workload?"`
  - `"@Kody, any blockers in the team’s workflow?"`
  - `"@Kody, provide a summary of our recent sprint issues"`

- **Sprint and Agile Insights**:

  - `"@Kody, what was our sprint velocity last week?"`
  - `"@Kody, how is the sprint completion rate this month?"`
  - `"@Kody, show the lead time for recent tasks"`

- **DORA Metrics**:

  - `"@Kody, show the DORA metrics for the past month"`
  - `"@Kody, how is our deployment frequency looking?"`
  - `"@Kody, what’s our current change failure rate?"`

- **Identify Bottlenecks**:
  - `"@Kody, where are the bottlenecks in our workflow?"`
  - `"@Kody, suggest improvements to reduce delays in our process"`

These specific commands allow you to quickly access detailed insights and initiate Kody’s functionalities in a targeted manner.

## How Kody Responds to Questions

To respond to user questions and interactions, Kody leverages various tools and data sources. You can review the full list of tools used by Kody in the **Capabilities** section.

## Notes on Using Kody in Communication Tools

- **Limitations**: Some advanced UI functionalities, like detailed dashboards, are only available on the web version of Kody.
- **Language**: Kody’s responses are provided in the language selected in your configuration settings.

For more complex workflows, consider accessing Kody through the web interface to take full advantage of its features.
