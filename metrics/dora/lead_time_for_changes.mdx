## Como calculamos
Através da sua ferramenta de gestão de código fonte, pegamos todos os deploys realizadas em um determinado período e buscamos o primeiro commit de cada deploy. 
A partir disso, calculamos o tempo entre o primeiro commit até o Deploy.

<Warning>Para o Lead Time For Changes funcionar, você precisa configurar como identificamos o deploy do seu time na tela de configurações.</Warning>


## Estrutura de retorno
```javascript
{
    leadTimeForChanges: {
        average: number,
        percentiles: {
            p50, number
            p75: number,
            p95 number,
        }
    }
}
```
## Porque é importante
O Lead Time for Changes é um indicador de quão rapidamente você consegue enviar novas funcionalidades para a produção. Um lead time elevado pode indicar tamanhos de lotes muito grandes, revisões de código/QA demoradas ou longos tempos de espera no CI/CD.

## Benchmark
| Elite          | Bom              | Média                | Precisa de atenção  |
| -------------- | ---------------- | -------------------- | ------------------- |
| Menos de 1 dia | Entre 1 e 7 dias | Entre 7 dias e 1 mês | Maior que 1 mês     |   

## Como utilizar o Lead Time For Changes
Meça o Lead Time for Changes em combinação com outras métricas de implantação e métricas DORA para garantir um equilíbrio saudável entre velocidade e estabilidade na sua entrega.
