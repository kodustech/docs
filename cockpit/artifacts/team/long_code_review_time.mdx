title: 'Demora no tempo de Code Review'
---
## Como funciona

Analisamos o tempo desde a abertura do Pull Request até o seu merge. Se uma alta porcentagem dos PRs excederem o tempo de revisão definido, um alerta é gerado.

## Por que isso é importante?

Monitorar o tempo de revisão de código é crucial para identificar possíveis gargalos no fluxo de trabalho. Revisões de código demoradas podem atrasar o ciclo de desenvolvimento e impactar negativamente a eficiência da equipe. Este artefato ajuda a identificar e melhorar a velocidade das revisões de código, garantindo que os PRs sejam analisados e mergeados em tempo hábil.

## Benefícios

1. **Identificação de gargalos:** Detectar revisões demoradas ajuda a identificar e resolver gargalos no fluxo de trabalho.
2. **Eficiência do processo:** Melhorar a velocidade das revisões aumenta a eficiência do ciclo de desenvolvimento.

## Dicas para acelerar revisões de código

- **Revisores dedicados:** Designe revisores dedicados para garantir que PRs sejam revisados rapidamente.
- **Limite de tempo:** Estabeleça limites de tempo claros para revisões de código.
