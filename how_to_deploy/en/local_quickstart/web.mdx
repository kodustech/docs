## Prerequisites

- Node.js (LTS version)
- Yarn or NPM
- Docker
- **Kodus API running** (see [Orchestrator Setup](/how_to_deploy/en/local_quickstart/orchestrator))
- LLM API Keys

## Running the project

<Tabs>
  <Tab title="Automated Setup (Recommended)">
    For a quick and automated setup, use our setup script:

    ```bash
    git clone https://github.com/kodustech/web.git
    cd kodus-web
    yarn dev:first-run
    ```

    This script will automatically:
    - ✅ Check if `.env.example` exists
    - ✅ Generate secure NextAuth secret
    - ✅ Handle JWT secret synchronization with your API
    - ✅ Create and configure the `.env` file
    - ✅ Provide clear next steps

    <Info>
      **JWT Secret Options:**
      - **Option 1**: Use existing API JWT secret (recommended if API is already configured)
      - **Option 2**: Generate new JWT secret (copy to API .env file)
      - **Option 3**: Skip JWT setup for now
    </Info>

    After running the script, you only need to:
    ```bash
    yarn install
    yarn docker:start
    ```

  </Tab>
  <Tab title="Manual Setup">
    If you prefer to configure everything manually:

    ### 1. Clone the repository

    ```bash
    git clone https://github.com/kodustech/kodus-web.git
    ```

    ### 2. Install dependencies

    ```bash
    yarn install
    ```

    ### 3. Configure environment variables

    ```bash
    cp .env.example .env
    ```

    ### 4. Generate Security Keys

    Before configuring your environment variables, you'll need to generate secure keys:

    ```bash
    # Generate NextAuth secret
    openssl rand -base64 32

    # Generate JWT secret (should match your API)
    openssl rand -base64 32
    ```

    You'll need to configure these variables in your `.env`:
    - `WEB_NEXTAUTH_SECRET` - NextAuth.js secret key
    - `WEB_JWT_SECRET_KEY` - JWT secret (must match your API's `API_JWT_SECRET`)

    ### 5. Configure API Connection

    Ensure your web application can connect to the Kodus API:

    ```env
    # API Configuration
    NEXT_PUBLIC_API_URL=http://localhost:3001
    NEXT_PUBLIC_API_HOST=localhost:3001
    ```

    ### 6. Configure Authentication

    Set up authentication providers in your `.env`:

    ```env
    # NextAuth Configuration
    NEXTAUTH_URL=http://localhost:3000
    NEXTAUTH_SECRET=your_generated_secret

    # JWT Configuration
    WEB_JWT_SECRET_KEY=your_generated_jwt_secret
    ```

    <Warning>
      **Important**: The `WEB_JWT_SECRET_KEY` must match the `API_JWT_SECRET` in your API's `.env` file for authentication to work properly.
    </Warning>

    ### 7. Start the development environment

    Launch the development server:

    ```bash
    yarn docker:build
    yarn docker:start
    ```

    The application will be available at `http://localhost:3000`

  </Tab>
</Tabs>

## Development workflow

### Basic Local Development

1. **Start services**: `yarn docker:start`
2. **Access web app**: `http://localhost:3000`
3. **Monitor logs**: `yarn docker:logs`

### Development with API Integration

If you're developing with the Kodus API:

1. **Ensure API is running**: The API should be accessible at `http://localhost:3001`
2. **Verify JWT secrets match**: Both web and API must use the same JWT secret
3. **Test authentication**: Try logging in to verify the connection

## Service endpoints

Access the application at:
- Development: `http://localhost:3000`
- API (external): `http://localhost:3001`

## Troubleshooting

### Common Issues

**Authentication not working:**
- Verify JWT secrets match between web and API
- Check if API is running and accessible
- Ensure `NEXT_PUBLIC_API_URL` is correct

**Build failures:**
- Clear Docker cache: `docker system prune -a`
- Rebuild: `yarn docker:build --no-cache`

**Environment variables not loading:**
- Verify `.env` file exists in project root
- Check for syntax errors in `.env` file
- Restart Docker services after `.env` changes

**API connection issues:**
- Verify API is running on port 3001
- Check network connectivity between containers
- Verify API health endpoint: `http://localhost:3001/health`

### Getting Help

If you encounter issues:
1. Check the logs: `yarn docker:logs`
2. Verify API status: Ensure the Kodus API is running
3. Check environment variables: Ensure all required values are set
4. Review the [API setup guide](/how_to_deploy/en/local_quickstart/orchestrator) for backend configuration
